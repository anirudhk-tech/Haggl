### 3. Preferred Vendor Feature (Detailed)

**Purpose:** Auto-fill vendor from past orders to enable quick reordering

**Behavior:**
1. When user adds an ingredient they've ordered before:
   - Checkbox "Preferred Vendor" appears and is auto-checked
   - Last vendor used is auto-filled with details:
     - Vendor name
     - Last order date
     - Last price per unit
     - Quantity ordered
     - Rating (if available)

2. User can:
   - Keep preferred vendor (skip sourcing for this item)
   - Uncheck to let Haggl find best price (may switch vendors)

3. If no previous order exists:
   - Checkbox is unchecked by default
   - Label shows "No previous vendor - Haggl will find best price"

**Database Query:**
```javascript
// Get preferred vendor for ingredient
db.order_history.findOne(
  {
    user_id: userId,
    ingredient: { $regex: ingredientName, $options: "i" },
    status: "completed"
  },
  { sort: { timestamp: -1 } }
)
```

**UI States:**

| State | Checkbox | Vendor Display |
|-------|----------|----------------|
| Has previous vendor | Checked (default) | Vendor card with details |
| No previous vendor | Unchecked (disabled) | "New ingredient - will source" |
| User unchecks | Unchecked | "Haggl will find best price" |

### 7. Vendor Evaluation with Preference Learning

**Purpose:** Each vendor displays an interactive radar chart with 4 evaluation parameters. Business owners can click UP/DOWN on each parameter to train the evaluation model on their preferences. Over time, the model learns what matters most to the business and curates vendor selection accordingly.

**How It Works:**
1. Each vendor shows a radar chart with 4 dimensions: Quality, Affordability, Shipping, Reliability
2. Each axis has UP (â–²) and DOWN (â–¼) buttons that act as **loss function inputs**
3. Clicking adjusts preference weights: UP increases importance, DOWN decreases it
4. The Voyage AI model learns these preferences through contrastive learning
5. Future vendor searches are automatically curated based on learned preferences

**Model Learning Details:**

The evaluation agent uses **Voyage AI embeddings** with contrastive preference learning:

1. **Training Data:** 200+ preference triplets (query, positive_vendor, negative_vendor)
2. **Focus Areas:** 40% quality-focused, 40% reliability-focused, 20% mixed
3. **Preference Vectors:** Learned direction vectors for each parameter
4. **Scoring:** Cosine similarity between vendor embedding and preference vectors
5. **Final Score:** `0.3 * embedding_score + 0.7 * weighted_parameter_score`

Over time, as the business owner clicks UP/DOWN:
- Weights adjust incrementally (learning rate: 5%)
- Vendor rankings automatically re-sort
- The model "understands" what the business values most
- Future vendor searches are pre-filtered to match preferences

## Preferred Vendor Logic (Technical)

```typescript
interface PreferredVendor {
  vendorId: string;
  vendorName: string;
  lastOrderDate: Date;
  lastPrice: number;
  lastQuantity: number;
  unit: string;
  rating: number;
  orderCount: number;
}

async function getPreferredVendor(
  userId: string,
  ingredient: string
): Promise<PreferredVendor | null> {
  const lastOrder = await db.collection("order_history").findOne(
    {
      user_id: userId,
      ingredient: { $regex: ingredient, $options: "i" },
      status: "completed"
    },
    { sort: { timestamp: -1 } }
  );

  if (!lastOrder) return null;

  // Get vendor stats
  const vendorStats = await db.collection("order_history").aggregate([
    {
      $match: {
        user_id: userId,
        vendor_id: lastOrder.vendor_id
      }
    },
    {
      $group: {
        _id: "$vendor_id",
        orderCount: { $sum: 1 },
        avgRating: { $avg: "$rating" }
      }
    }
  ]).toArray();

  return {
    vendorId: lastOrder.vendor_id,
    vendorName: lastOrder.vendor,
    lastOrderDate: lastOrder.timestamp,
    lastPrice: lastOrder.price_per_unit,
    lastQuantity: lastOrder.quantity,
    unit: lastOrder.unit,
    rating: vendorStats[0]?.avgRating || 0,
    orderCount: vendorStats[0]?.orderCount || 1
  };
}
```

## Mobile Responsiveness

The dashboard should be fully responsive:

- **Desktop (>1024px):** Full dashboard with side navigation
- **Tablet (768-1024px):** Collapsible navigation, stacked cards
- **Mobile (<768px):** Bottom navigation, single-column layout, SMS-first experience

## Design Inspiration

Based on modern B2B SaaS dashboards:

1. **Stripe Dashboard** - Clean, data-focused, excellent tables
2. **Linear** - Minimal, fast, keyboard-first
3. **Notion** - Flexible, blocks-based, intuitive
4. **Mercury** - Banking-style, trust-inspiring, clear hierarchy

Key principles:
- **Data density** - Show relevant info without clutter
- **Clear CTAs** - Primary actions always visible
- **Status visibility** - Order progress always clear
- **Quick actions** - Reorder, approve, deny in one click
